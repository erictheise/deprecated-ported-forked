By and large, the "alphabet" of this language consists of electrical pulses, and the "words" are made up of sequences of such pulses arranged according to standard codes. If we designate the absence of a pulse as zero and the present of a pulse as one, we can encode almost any kind of information we want into groups of one's and zero's.

Generations of Programming Languages

  • 1GL: 0s and 1s via switches and patchcords
  • 2GL: assembly language; readable, but chip-specific
  • 3GL: high-level; FORTRAN, ALGOL, COBOL (1950s) up to the present day; machine-independent
  • 4GL: domain-specific languages
  • 5GL: artificial intelligence
Third generation languages are what we usually think of as programming languages. 4GL & 5GL, like VR, are terms that have been distorted by hype and unreasonable expectations. 4GL as report generator or modeling language like GAMS.

hello, world

Introduced to a "wide" audience in The C Programming Language (1978), "hello, world" has become the traditional first program to write in a language. Its stated purpose is to output that phrase to an appropriate output device, but it also reveals something of a language's syntax and structure. 


main( ) {
  printf("hello, world");
}
Hello world is especially satisfying when creating your first map and you actually get a world.

Python. Beer.


#!/usr/bin/env python
# -*- coding: iso-8859-1 -*-
"""
99 Bottles of Beer (by Gerold Penz)
"""

for quant in range(99, 0, -1):
   if quant > 1:
      print quant, "bottles of beer on the wall,", quant, "bottles of beer."
      if quant > 2:
         suffix = str(quant - 1) + " bottles of beer on the wall."
      else:
         suffix = "1 bottle of beer on the wall."
   elif quant == 1:
      print "1 bottle of beer on the wall, 1 bottle of beer."
      suffix = "no more beer on the wall!"
   print "Take one down, pass it around,", suffix
   print "--"
Line by line. Iteration (for), variable (quant), function (range), externality (print), flow control (if, else, elif). What's hidden? Python is indentation based. Carling McManus: speed.

Branching


def is_on?(light)
  if light.on?
    true
  else
    false
  end
end

def is_on?(light)
  unless light.on?
    false
  else
    true
  end
end

def is_on?(light)
  light.on? ? true : false
end

def is_on?(light)
  true if light.is_on?
  false if !light.is_on?
end
Direct port. Use of negation. Use of ternary. Natural English? You may fail an interview by writing non-idiomatic code. Sometimes called "syntactic sugar", i.e., empty calories.

perl Poetry

Many Perl programmers are linguistically adept; the expressivity and flow of our language attracts people who enjoy the written word. Perl programmers also tend to have a lot of free time from getting their jobs done so quickly, and so it's natural that they sometimes blend Perl and wordplay. One common manifestation of this whimsy is Perl poetry: a poem that also happens to be a functioning program.
